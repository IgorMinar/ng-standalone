Angular Standalone Component Demo

This is a simple demo project, that shows what a standalone @Component, @Directive, and @Pipe in Angular could look like in the future.

The mental model of standalone Components, Directives, and Pipes is simple:

  • the class of each component, directive, and pipe (from now on just entities) is shadowed by a "virtual" NgModule that is automatically created by the system.
  • this NgModule declares only the single entity that it shadows
  • any existing NgModule or API that accepts NgModules can import the standalone entity directly
  • a standalone entity can import any other standalone entity, or an existing NgModule

This design has enables strong introp and encapsulation guarantees making it easy to mix and match standalone entitities with code of existing applications and libraries in the Angular ecosystem.

Specifically, this design has the following properties:

  • encapsulation and isolation:
    • a standalone entity is available to be used in a template of a non-standalone component only if it is imported into the NgModule of the non-standalone component
    • a standalone entity can use only other entities that were defined in its imports
  • error resiliency:
    • importing a non-standalone entity directly anywhere in the system is an error
    • declaring a standalone entity in an NgModule is an error
  • discovery and readability:
    • to make it clear to both the code reader, as well as the compiler/IDE/linter which entity is standalone, we use syntatic element in the form of standalone property within the decorator metadata
  • ability to evolve the API in the future
    • if the "standalone" style is accepted by the community, the standalone property could default to true and become optional

Demo implementation notes: The demo is built by wrapping an existing @Component, @Directive, and @Pipe decorators and generating a hidden (virtual) NgModule. The application, including the standalone entities, is then compiled using the JIT compiler.
