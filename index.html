Angular Standalone Component Demo

This is a simple demo project, that shows what a standalone @Component, @Directive, and @Pipe in Angular could look like in the future.

The mental model of standalone Components, Directives, and Pipes is simple:

  • each component, directive, and pipe class is shadowed by a "virtual" NgModule that is automatically created by the system.
  • this NgModule declares only the single entity that it shadows
  • any existing NgModule or API that accepts NgModules can import the standalone entity directly
  • a standalone entity can import any other standalone entity, or an existing NgModule
  • importing a non-standalone entity directly anywhere in the system is an error
  • declaring a standalone entity in an NgModule is an error
  • to make it clear to both the code reader, as well as the compiler which entity is standalone, we use syntatic element in the form of standalone property within the decorator metadata
  • in the future, if the "standalone" style is accepted by the community, the standalone property could default to true and become optional

Demo implementation notes: The demo is built by wrapping an existing @Component decorator and generating a hidden (virtual) NgModule. The application, including the standalone component, is then compiled using the JIT compiler.
